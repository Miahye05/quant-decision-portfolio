# Portfolio Principles

## 1. Decision-first, not tool-first

Each project should start with a decision problem. Tools are included only because they help answer that problem.

## 2. Public means safe

Never publish internal files, private data, API keys, access tokens, unreleased reports, or materials created by other people without permission.

## 3. AI assistance must be explainable

AI can help with coding, debugging, and refactoring. It cannot replace understanding. If I cannot explain a block of code, it is not ready for the portfolio.

## 4. Limitations are part of credibility

A strong project explains what it can show and what it cannot show.

## 5. Small and reproducible beats large and vague

A small notebook with clear data, assumptions, and results is better than a large project that cannot be explained.
